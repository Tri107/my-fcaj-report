**On this page**, I will introduce the **Worklog**, which documents the detailed process of learning, exploring, and practicing services on the AWS cloud platform.

The internship program was completed over **12 weeks** (about 3 months). During this time, I progressed from getting familiar with basic concepts to building and deploying a real-world project on AWS. The detailed weekly contents are as follows:

**Week 1:** [Getting familiar with AWS, account setup, and basic security services (IAM, MFA)](1.1-week1/)

**Week 2:** [Learning and practicing AWS networking (VPC, Subnet, Route table, EC2)](1.2-week2/)

**Week 3:** [Configuring Route 53, VPC Peering, Transit Gateway, and EC2 Auto Scaling](1.3-week3/)

**Week 4:** [Exploring storage services (S3, Storage Gateway, CloudFront) and static websites](1.4-week4/)

**Week 5:** [Deep dive into Security, Identity (IAM roles, KMS), CloudTrail, and Athena](1.5-week5/)

**Week 6:** [Exploring Databases (RDS, DynamoDB), Data Lake, and using AWS CLI](1.6-week6/)

**Week 7:** [Serverless computing (Lambda, API Gateway) and system architecture design](1.7-week7/)

**Week 8:** [Building CloudFormation/SAM templates and programming Lambda for the final project](1.8-week8/)

**Week 9:** [Finalizing the deployment of the basic system, hosting the user interface with AWS Amplify](1.9-week9/)

**Week 10:** [Integrating Amazon Textract for automated CV evaluation and setting up AWS WAF](1.10-week10/)

**Week 11:** [Testing, optimizing the system and editing the interface, preparing the report](1.11-week11/)

**Week 12:** [Finalizing the report and presenting the internship summary report](1.12-week12/)
